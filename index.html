1) ¿Qué versiones de JavaScript hay en el mercado?

  1. JavaScript 1: Creado junto con Netscape 2.0, aguanta una gran cantidad de instrucciones y funciones, todas estas ya introducidas en su versión estándar.
  2. JavaScript 1.1: Versión diseñada con el lanzamiento de navegadores 3.0. Es muy similar a su version anterior, sólo se le incorporó el procesamiento dinámico de imágenes y la creación de matrices.
  3. JavaScript 1.2: Versión diseñada con el lanzamiento de los navegadores 4.0. Tenía como desventaja que era un poco diferente en las plataformas de Microsoft y Netscape, porque los dos navegadores habían evolucionado de manera diferente y ambos competían por el mercado.
  4. JavaScript 1.3: Versión diseñada para navegadores 5.0. Ya con esta versión no había diferencias entre plataformas.
  5. JavaScript 1.5: Versión actual de JavaScript.
  6. ECMAScript 5: Versión lanzada en 2009, funciona en todos los navegadores.
  7. ECMAScript 6: Versión lanzada en 2015, tuvo muchas mejoras importantes, la mayoría de desarrolladores profesionales la utilizan, funciona en todos los navegadores, menos en Internet Explorer.
  8. ECMAScript 7: Versión lanzada en 2016, actualización de ES6. Se incorporó: El método Array.prototype.includes() y el operador exponencial (**).
  9. ECMAScript 8: Versión lanzada en 2017, conocida como ECMAScript 2017. Se le incorporó: async/await, las cuales funcionan usando generadores y promesas.
  10. ECMAScript 9: Versión lanzada en 2018, conocida como ECMAScript 2018. Se le incorporó: operadores rest/spread para variables (tres puntos: ...identificador), iteracion asincronica, Promise.prototype.finally().
  11. ECMAScript 10: Versión lanzada en 2019, conocida como ECMAScript 2019. Se le incorporó: Array.flat(), Array.flatMap(), String.trimStart(), String.trimEnd(), errores opcionales en el bloque catch, Object.fromEntries(), Symbol.description.
  12. ECMAScript 11: Versión lanzada en 2020, conocida como ECMAScript 2020. Aparte de tener nuevas funciones, a esta versión se le añadieron tipos primitivos BigInt para enteros primitive, el operador de unión nula (nullish coalescing), encadenamiento opcional (optional chaining) y el objeto globalThis.
  13. ECMAScript 12: Versión lanzada en 2021, conocida como ECMAScript 2021. Es la última versión de JavaScript, diseñada en github por la compañía ECMA International. Se le incoporaron las siguientes funciones: Operador de asignación lógica, método .replaceAll(), métodos de clase privados, getters y setters privados, promise.any(), aggregateError y weakRef.

2) ¿Cómo se integra JavaScript a HTML?

Hay 2 formas de integrar JavaScript a HTML.
La primera es con el atributo "src" dentro de la etiqueta <script>, se recomienda agregarlo en la sección <head> para separlo de nuestro contenido HTML.
Este atributo nos permite enlazar la ruta de nuestro documento personalizado .js.
Por ejemplo: <script src="js/script.js"></script>.
O también podemos enlazar documentos externos de un tercero de la siguiente manera:
<script src="https://kit.fontawesome.com/c28dc5c839.js"></script>.
La segunda manera es aplicando nuestro código js dentro la etiqueta <script></script> dentro de la sección <body>.
Esta manera la utilizaremos si queremos mostrar nuestro contenido js dentro del cuerpo de nuestra página web.

3) ¿Cómo es el manejo de las variables, tipos de datos y expresiones regulares en JS?

Variables: En JavaScript para declarar variables se utilizan las palabras clave var, const y let.
Estas variables las utilizamos para guardar y registrar datos para refirnos a ello a lo largo de nuestro código.
Var: Esta palabra nos permite declarar variables y adicionalmente le podemos inicializar un valor.
Por ejemplo:
var edad = 10;.
console.log(edad); // salida "10".

Let: Esta palabra nos permite declarar variables limitadas por un bloque. Es decir, todo lo que está dentro de llaves {} es un bloque.
Por ejemplo:
if (true) {
let nombre = Lucas;
console.log(nombre); // salida "Lucas"
}

Const: Esta palabra nos permite declarar variables con valores constantes. Es muy parecido a let, ya que solamente
podemos acceder a nuestras declaraciones const dentro del bloque en el que fueron declaradas.
La diferencia con let es que const no puede modificarse ni volver a declararse, entonces toda declaración const, debemos inicializada en el momento de la declaración.
Por ejemplo:
const nombre = {
mensaje: "Lucas",
edad: 18
}

Tipos de datos:: La última versión de ECMAScript define nueve tipos de datos:

  1. Undefined: Solo se asigna automáticamente el valor inicial de las variables declaradas o parámetros formales, sin los parámetros reales.
    Se asigna de la siguiente manera:
    typeof instance === "undefined"
  2. Boolean: Dato lógico que puede tener solamente valores "true" o "false".
    Se asigna de la siguiente manera:
    typeof instance === "boolean"
  3. Number: Tipo de dato numérico.
    4. Se asigna de la siguiente manera:
    typeof instance === "number"
  4. String: Secuencia de caracteres utilizada para representar un texto.
    Se asigna de la siguiente manera:
    typeof instance === "string"
  5. BigInt: Tipo de dato numérico que representa números enteros en el formato de precision arbitrario.
    Se asigna de la siguiente manera:
    typeof instance === "bigint"
  6. Symbol: Este tipo representa un identificador único.
    Se asigna de la siguiente manera:
    typeof instance === "string"
  7. Null: Valor nulo declarado, comúnmente de manera intencional, a un inexistente objeto o dirección, si el objeto no se ejecuta, se muestra null.
    Se asigna de la siguiente manera:
    typeof instance === "null"
  8. Object: Este tipo se aplica a una estructura de datos que contiene datos e instrucciones para trabajar con esos datos. Por lo general siempre se refiere a cosas del mundo real.
    Se asigna de la siguiente manera:
    typeof instance === "object"
  9. Function: Parte del código que puede ser llamada por otro código, o por una variable que llame a la función.
    Se asigna de la siguiente manera:
    typeof instance === "function"
Expresiones Regulares: Es un sistema para buscar, capturar o reemplazar texto mediante patrones, las expresiones regulares son objetos.
Estos patrones se utilizan con los métodos:
  • exec(): Ejecuta una coincidencia en una cadena. Devuelve una matriz de datos o null si no hay ninguna coincidencia.
  • test(): Ejecuta una coincidencia en una cadena. Devuelve una salida true o false.
  • match(): Devuelve una matriz de todas las coincidencias, incluidos los grupos de captura, o null si no se encontró ninguna coincidencia.
  • matchAll(): Devuelve una salida que contiene todas las coincidencias, incluidos los grupos de captura.
  • search(): Intenta una coincidencia en una cadena. Devuelve el índice de esa coincidencia, o -1 si no coincide.
  • replace(): Realiza una búsqueda coincidente en una cadena y reemplaza la subcadena coincidente con otra subcadena de reemplazo.
  • replaceAll(): Realiza una búsqueda de todas las coincidencias de una cadena y reemplaza la subcadena coincidente con otra subcadena de reemplazo.
  • split(): Divide una cadena en una serie de subcadenas usando una expresión regular o una cadena fija.

4) ¿Cómo son todas las formas de declaración de funciones en JS?

Hay 7 formas de declarar una función en JavaScript:

  1. Function declaration: Esta función se creará utilizando la palabra function, seguida de un nombre que identifique nuestra función y
    un conjunto de parámetros entre paréntesis () y llaves {}. Lo cual determinará el contenido de nuestro conjunto de sentencias.
    Por ejemplo:
    function hola(nombre){
    console.log("Hola ${nombre}.")
    }
    hola("Lucas"); // salida Hola Lucas.
  2. Function expression: Similar a function declaration, la única diferencia es que esta función no comienza con una declaración function
    y el nombre de la función es opcional.
    Las funciones de este tipo se pueden almacenar en una variable.
    Por ejemplo:
    const suma = function sumaTres(valor) {
    return valor + 3;
    }
    console.log(sumatres(7)); // salida 10.
  3. IIFE (Immediately Invoked Function Expression): Cuando definimos una función es porque la llamaremos varias veces, pero si queremos
    llamar a la función solo una vez y obtener el resultado, podemos utilizar la función IIFE. Estos tipos de funciones se realizan de inmediato y no se puede acceder luego.
    Por ejemplo:
    ( function () {
    let nombre = "Lucas"
    return "Hola ${nombre}"
    })() // salida Hola Lucas
  4. Shorthand method definition: Para crear esta función, necesitamos darle un nombre a la función luego, una lista de parámetros
    entre paréntesis () y llaves {} para encerrar el cuerpo de instrucciones.
    Por ejemplo:
    const gente = {
    personas: [],
    add(...personas) {
    this.personas.push(...personas);
    },
    saluda(index) {
    return "Hola ${this.personas[index]}";
    }
    };
    gente.add('Lucas', 'Álvaro', 'José');
    gente.saluda(1) // salida Hola Álvaro.
  5. Arrow function: Además de que la conexión es diferente a las anteriores, este tipo de función tiene las siguientes características.
    • Esta función no crea su propio contexto cuando se ejecuta. A diferencia de "function expression" o "function declaration" que si crean su propio contexto.
    • Esta función es anónima.
    • Si no utilizamos llaves {} al definir nuestra función, esta misma nos dará como resultado la ejecución de instrucciones que hayamos introducido.
    Ejemplo:
    const saluda = (nombre) => {
    return "Hola ${nombre}";
    }
    console.log(saluda("Lucas")) // salida Hola Lucas.
  6. Generator function: La característica más notable de esta función, es que nos permite pausar o finalizar la función en algún punto
    de la ejecución de instrucciones compuesta por la función, y luego reanudarla desde donde la detuvimos anteriormente.
    Solo necesitamos utilizar el símbolo * al principio de la definición para indicar esta función.
    Por ejemplo:
    function *generatorFunction()
    {
    console.log("Ejemplo generator Function.");
    yield 'Hola';
    console.log('¿Continuar?');
    yield 'Fin del ejemplo';
    }
    const gen = generatorFunction();
    console.log(gen.next().value);
    console.log('Función pausada');
    console.log(gen.next().value);
    console.log(gen.next().value);

    Salida:
    // Ejemplo generator Function.
    // Hola
    // Función pausada
    // ¿Continuar?
    // Fin del ejemplo
    // undefined
  7. Function constructor: Función bastante inusual, esta función crea un nuevo objeto function. Entonces podemos crear
    una variable que llame al objeto de función. Cuando llamamos a este objeto, podemos pasar cualquier parámetro que queramos.
    Por ejemplo:
    const sumaFunction = new Function('num1', 'num2',
    'return num1 + num2'
    );
    console.log(sumaFunction(8, 12)) // salida 20.

5) ¿Cuáles son los operadores y conversión entre tipos?

Operadores: Los operadores nos permiten manejar los valores de nuestras variables, hacer operaciones matemáticas con esos valores y comparar variables.

Tipos de Operadores:

  1. Asignación: Es el operador más usado y el más fácil. Se usa para guardar valores específicos dentro de una variable.
    Su símbolo es: =.
    Ejemplo:
    var num1 = 10;
  2. Incremento y decremento: Operadores que sólo sirven para asignar variables numéricas.
    Se usan para incrementar o decrementar en una unidad el valor de nuestra variable.
    (Incremento) Su símbolo es: ++.
    Ejemplo:
    var num1 = 9;
    ++num1;
    alert(num1); // num1 = 10
    (Decremento) Su símbolo es: --.
    Ejemplo:
    var num2 = 6;
    --num2;
    alert(num2); // num2 = 5
  3. Lógicos: Operadores utilizados para determinar qué instrucciones debe ejecutar el programa según ciertas condiciones.
    Símbolos:
    • +, -, *, /: Suma, resta, multiplicación y divisón.
    • !: Negación.
    • &&: Operador and (y).
    • ||: Operador or (o).
  4. Relacionales: Operadores parecidos a los que definimos para las matématicas.
    Símbolos:
    • <: Menor que.
    • <=: Menor o igual.
    • >: Mayor que.
    • >=: Mayor o igual.
    • ==: Igual que.
    • !=: Distinto de.

conversiones de tipo: Los operadores y funciones transforman de manera automática los valores que se les pasan de la manera correcta. A esto se le llama "conversión de tipo".

Las tres conversiones de tipo:

  1. ToString: Ocurre cuando se manifiesta un valor en forma de texto.
    Ejemplo:
    let value = false;
    alert(typeof value);
    value = String(value); // value pasa a ser el string "false"
    alert(typeof value); // string
  2. ToNumber: Ocurre en conversiones numéricas.
    Ejemplo:
    let a = "55";
    alert(typeof a); // string
    let num = Number(a); // se convierte en 55
    alert(typeof num); // num = 55
  3. ToBoolean: Ocurre con las operaciones lógicas.
    Ejemplo:
    var falso = new Boolean(false); // valor inicial de false
    var a = Boolean(falso); // valor inicial de true
    var s = new String("Hola"); // objeto string
    var b = Boolean(s); // valor inicial de true

6) ¿Cuáles son las estructuras de control de JS?

Las estructuras de control de JavaScript son:

  • IF: Una declaración if especifica un bloque de código para ejecutar si nuestra condición es verdadera.
    Ejemplo:
    if(condición){
    // Instrucciones si la condición es verdadera
    }
  • IF/ELSE: Una declaración if especifica un bloque de código para ejecutar si nuestra condición es verdadera y la opción de ejecutar otro bloque de código si la condición no es verdadera.
    Ejemplo:
    if(condición){
    // Instrucciones si la condición es verdadera
    }else{
    // Instrucciones si la condición es falsa
    }
  • SWITCH: Se usa para realizar distintas acciones en distintas condiciones.
    Ejemplo:
    switch(valor){
    case valor1:
    // Instrucciones para el valor1
    break;
    case valor2:
    // Instrucciones para el valor2
    break;
    case valor3:
    // Instrucciones para el valor3
    break;
    default:
    // Instrucciones en caso de que no se cumpla ninguna condición
    }
  • FOR: Nos permite ejecutar el código un número definido de veces.
    Ejemplo:
    for(inicialización; condición; actualización){
    // Código que se repite
    }
  • WHILE: Permite ejecutar un bloque de código si se cumplen las condiciones señaladas entre paréntesis.
    Ejemplo:
    while(condición){
    // Código a ejecutar
    }
  • DO/WHILE: Su diferencia con while, es que necesariamente debe ejecutarse 1 vez, luego puede ejecutarse las veces que las condiciones lo indiquen.
    Ejemplo:
    do{
    // Código a ejecutar
    }while(condición); // Hasta que...
  • Sentencias break y continue: Manipulan la secuencia de un programa.
    Break: Salir o saltar un bucle, finalizándolo.
    Ejemplo:
    var i = 0;
    for(i = 0; i < 10; i++){
    if (i == 5){ // Cuando i llega a 5 se ejecuta el break y sale del bucle for
    break;
    }
    Continue: Saltar hasta la siguiente iteración de un bucle o más.
    Ejemplo:
    var i = 0;
    for(i = 0; i < 10; i++){
    if (i == 5 || i == 7){
    continue; // Saltar iteración si i vale 5 o 7 (no ejecutará el alert)
    }
  • FOREACH: Ejecuta la función indicada una vez por cada elemento presente, en orden ascendente de un array.
    Ejemplo:
    const array = ['a', 'b', 'c'];
    array.forEach(elemento => console.log(elemento));
    // Salida: "a"
    // Salida: "b"
    // Salida: "c"

7) ¿Qué es el objeto dom en JS? Y ¿Cómo trabaja?

DOM: Document Object Model. Cuando hablamos de DOM, nos estamos refiriendo a una estructura o interfaz de programación, que nos permite crear, modificar, cambiar o eliminar los elementos de nuestro documento desde JavaScript. O también podemos añadir eventos a nuestros elementos para hacer nuestra página más dinámica. Esta interfaz trabaja vinculando JavaScript con nuestro código fuente de HTML, lo que hace que JavaScript tenga acceso a cualquier elemento de nuestro documento HTML y ejecute todas sus funciones en elementos HTML individuales. Esta combinación HTML-JS sirve para darnos un diseño de página web interactivo.

8) ¿Qué función tiene en JS el Object Map?

Object Map es una definición de clave-valor en la que se puede utilizar cualquier tipo como clave. Esta es una de las propiedades más importantes de los mapas y su principal diferencia con los objetos, donde las claves solo pueden ser cadenas de texto. Este método cumple la función de aplicar cambios a nuestros elementos, ya sea una operación matemática o cualquier otro tipo de operación que necesitemos realizar.

9) ¿Cuáles son todos los eventos que trabaja JS?

Los eventos que trabaja JavaScript son:

  • onblur: Un elemento pierde el foco.
  • onchange: Un elemento ha sido modificado.
  • onclick: Pulsar y soltar el mouse.
  • ondblclick: Pulsar dos veces seguidas con el mouse.
  • onfocus: Un elemento obtiene el foco.
  • onkeydown: Pulsar una tecla y no soltarla.
  • onkeypress: Pulsar una tecla.
  • onkeyup: Soltar una tecla pulsada.
  • onload: Página cargada completamente.
  • onmousedown: Pulsar un botón del mouse y no soltarlo.
  • onmousemove: Mover el mouse.
  • onmouseout: El mouse "sale" del elemento.
  • onmouseover: El mouse "entra" en el elemento.
  • onmouseup: Soltar el botón del mouse.
  • onreset: Inicializar un formulario.
  • onresize: Modificar el tamaño de la ventana.
  • onselect: Seleccionar un texto.
  • onsubmit: Enviar el formulario.
  • onunload: Se abandona la página, por ejemplo al cerrar el navegador.
